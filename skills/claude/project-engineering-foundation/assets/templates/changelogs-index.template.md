# Changelogs Index

> Scope: user-visible features, behavior changes, API changes, dependency upgrades, and project setup changes. Debug, performance, security, and review-driven fixes go in `ref/reviews/INDEX.md`.

## Naming

Use `CHANGELOG_X.md`, where `X` is the next integer aligned with existing changelog files. Before creating a changelog, run `ls ref/changelogs/` and choose one higher than the current maximum.

## File Structure

- Summary
- Changes grouped by module or layer
- Notes for tradeoffs, unresolved risk, related review, or omitted promotion/fix rationale
